================================================================================
  你只要 APK：按下面做（用 GitHub 免费云端替你打包）
================================================================================

【说明】你的电脑是 Windows，且没有 Linux / WSL / Docker 时，任何人都不可能
在本机“点一下就出 APK”。我已经在项目里加好了「云端自动打包」：由 GitHub 的
Linux 服务器运行 Qt 官方工具，把 APK 生成好，你只需下载。

--------------------------------------------------------------------------------
第一步：把本项目上传到 GitHub
--------------------------------------------------------------------------------
1. 打开浏览器，注册/登录 https://github.com
2. 新建一个仓库（例如名字填 ASR），不要勾选 “Add README”
3. 用下面任一方式把「整个 ASR 文件夹」上传到这个仓库：
   - 安装 “GitHub Desktop”，选 File → Add local repository，再 Publish；或
   - 网页上点 “uploading an existing file”，把文件夹里所有文件拖进去提交

确保上传后，仓库里能看到文件夹：.github/workflows/build-android-apk.yml

--------------------------------------------------------------------------------
第二步：启动打包
--------------------------------------------------------------------------------
1. 打开你的仓库网页
2. 点顶栏 “Actions”
3. 左侧点 “Build Android APK”
4. 右侧点 “Run workflow” → 再点绿色 “Run workflow”

（若你刚推送了 main/master 分支，它也会自动跑一遍。）

--------------------------------------------------------------------------------
第三步：下载 APK
--------------------------------------------------------------------------------
1. 等该次运行结束（可能要 30～90 分钟，第一次最慢）
2. 点开那一次运行记录，页面最下方找到 “Artifacts”
3. 下载 “asr-android-debug-apk.zip”，解压后即可得到 .apk，拷到手机安装

注意：这是调试包（debug），部分手机需在「安全」里允许安装未知来源应用。

--------------------------------------------------------------------------------
若打包失败（红色 X）
--------------------------------------------------------------------------------
本项目依赖语音识别 SDK、numpy、网络等，Android 打包比桌面版复杂，第一次
失败很常见。请点开失败的那一步，把日志最后几十行复制给会开发的人，或发给我，
再针对性改配置（例如 buildozer 依赖、权限等）。

更技术向的补充说明见：ANDROID_BUILD.txt
